React Redux介绍

正如What Does Redux Do?所描述的，Redux在一个单独的store中管理你应用的状态。然后你可以将状态部分抽离出来作为属性插入到你的组件中。这样可以让你在全局的某个地方管理你的数据，然后直接传递给应用中的任何一个组件，再也不需要一层一层操作来进行属性传递

为了更方便理解我们后续的步骤，建议先将redux添加到你的项目中

当然你也可以通过npm安装到项目中去。

redux vs react-redux

redux提供了一个store，让你通过它存取数据，并且当数据变化的时候作出响应——但它仅仅限于以上这些。实际上react-redux才提供了让你将React组件分散的状态连接起来的能力。

这两个库一般都是配套使用，大多数情况下，当大家在React中提到redux的时候，通常指的是这两个一起。本文创建的应用将展示一个redux以及react-redux一起使用的例子。我们先看下从组件中移除state的例子

提供一个Store

Redux为整个应用管理一份全局的状态，通过react-redux中Provider组件来封装整个应用，应用树中的每一个组件都可以通过连接到Redux的store，只要它需要。这意味着，App以及App的子组件，子组件的子组件基于connect封装后都可以连接到Redux的store中。

想象下，当在一堆煎饼上倒上糖浆，它怎么做到将糖浆渗透到所有煎饼上呢——即使你只是倒在最上面那个煎饼上。在redux中，Provider会为它做这些事情。
